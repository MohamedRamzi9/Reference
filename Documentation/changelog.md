### Changelog format

## <Id> <Date>
### Added:
- description of added features
### Fixed:
- description of fixed issues
### Changed:
- description of changed features
### Removed:
- description of removed features
### Refactored:
- description of refactored features
  
### Id
- The same unique identifier used in commit messages to reference the commit in the changelog.

### Date 
- The date of the commit in `DD-MM-YYYY` format.
